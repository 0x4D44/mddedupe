## Team model

- **Implementer** (opus) ‚Äî writes code TDD-first, keeps `cargo build/test/clippy/fmt` green.
- **Reviewer** (code-reviewer skill) ‚Äî reviews each stage's diff, writes a CR doc to `wrk_docs/`.
- **Devil's advocate** (opus) ‚Äî independently challenges the work: hunts for the
  weakest assumption, the untested edge, the spec gap. Runs in parallel with Reviewer.
- **Fixer** (code-review-apply-feedback skill) ‚Äî applies reviewer + devil's-advocate findings.
- **Verifier** (final) ‚Äî compares the finished implementation against the HLD; iterate to 100%.

## Plan (sequential stages; commit after each)

1. **Stage 1 ‚Äî Selection engine (neutral default).** `FallbackStrategy`, `KeepPolicy`
   (+ neutral `Default`), `SurvivorReason`, `select_survivors`; wire into
   `process_duplicates` / read-only display / `find_duplicates_in_dirs` via
   `&KeepPolicy`, callers pass `KeepPolicy::default()`. Output + len-1 accounting
   unchanged (correct under neutral). Deps: `globset`, dev `filetime`. Existing tests
   pass `default()`; new unit tests exercise protect + fallback directly.
2. **Stage 2 ‚Äî Accounting + explain + safe collapse.** Two-metric accounting
   (redundancy + removable) in `find_duplicates_in_dirs` / `process_duplicates` /
   `JsonSummary`; `KEPT (reason)` explain output both paths; protect-aware
   `collapse_group_by_identity`. Still neutral default ‚Üí CLI unchanged; tested via
   explicit policies.
3. **Stage 3 ‚Äî Config + CLI + flip default ON.** `toml` dep; `ConfigFile`/`KeepConfig`;
   `resolve_keep_policy`; flags `--protect-dir/--protect-name/--no-protect/--keep/--config`;
   `AppError` variants; `run_app` resolves convention default ‚Üí feature ON.
4. **Stage 4 ‚Äî Docs.** `ARCHITECTURE.md`, `CLAUDE.md`, `docs/usage.md`, `readme.md`.
5. **Final verification.** Agent team diffs implementation vs HLD; iterate to 100%.
