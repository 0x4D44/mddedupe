## Team model

- **Implementer** (opus) — writes code TDD-first, keeps `cargo build/test/clippy/fmt` green.
- **Reviewer** (code-reviewer skill) — reviews each stage's diff, writes a CR doc to `wrk_docs/`.
- **Devil's advocate** (opus) — independently challenges the work: hunts for the
  weakest assumption, the untested edge, the spec gap. Runs in parallel with Reviewer.
- **Fixer** (code-review-apply-feedback skill) — applies reviewer + devil's-advocate findings.
- **Verifier** (final) — compares the finished implementation against the HLD; iterate to 100%.
