## Journaled intentional deviations (treated COMPLETE)

1. **`GroupPlan` built once** replaces HLD §5.3's "4 on-demand mtime reads in 4 places" — eliminates display-vs-deleted divergence at the type level (`process_duplicates(&[GroupPlan])`). Sound improvement.
2. **Explain label shows nearest matching dir/file NAME** (e.g. `KEPT (protected: dir keepme)`), not the glob. More useful; HLD example showed the glob (`dir 0*`). Documented in S4.
3. **`--no-protect` ⇒ empty protects + `lexical`** (reconciled with HLD §3.3 "neutral/today"); an explicit `--keep` still wins. Matches HLD intent precisely.

## Minor observations (not gaps)

- Scenario 13's end-to-end config-file load is exercised at unit level (`resolve_keep_policy` with injected base_dir) rather than by writing a `.mddedupe.toml` and running the binary. The auto-discovery + parse + merge logic is fully and hermetically tested; the binary wiring is a one-line call (`resolve_keep_policy(&args, Path::new("."))`). No behavioral gap, but a binary-level config test would be the single highest-value addition if more coverage were ever wanted.
- 10 clippy warnings are all pre-existing test-helper lints (Rust 1.91 `io_other_error`, Copy-clone, complex-type); none introduced by this feature.
